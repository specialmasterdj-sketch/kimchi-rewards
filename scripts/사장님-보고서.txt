사장님께 — KIMCHI MART 멤버십 앱 운영 점검

한 줄 요약
앱 기능은 완성됐고 5만 7천 손님 데이터도 등록 완료입니다.
본격 사용 전에 두 가지를 처리해야 합니다.

────────────────────────────────

1. 보안 설정 (시급, 오늘 안)

문제: 손님 데이터 (폰번호, 이름, 포인트 5만 7천 명)가 잠금 없이
저장되어 있습니다. 누군가 시스템 주소만 알면 명단을 가져갈 수 있고
포인트도 임의로 변경 가능합니다.

비유: 가게 셔터를 안 내리고 출근한 상태입니다.

해결: Firebase Console에서 보안 규칙(Security Rules) 한 번
설정하면 끝. 손님 데이터를 셔터로 막는 작업입니다.

비용: 무료
소요: 5분

────────────────────────────────

2. Firebase 유료 플랜 전환 (이번 달 안)

현재: 무료 플랜. 월 다운로드 한도 10 GB.

5만 손님이 본격 사용 시:
- 매일 약 12 GB 다운로드
- 한 달 약 360 GB (한도의 36배 초과)
- 한도 초과 순간 앱 멈춤 (손님 못 들어감)

해결: Blaze 플랜 (사용량 기반) 전환.

예상 비용: 월 약 40~50만원 (300~400 USD)
신용카드 등록만 하면 자동 청구.
매출 대비: 0.04% (카드 수수료보다 적음)

참고: 1만 명 사용 시는 월 6~10만원, 점진 증가입니다.

소요: 5분 (카드 등록)

────────────────────────────────

3. 운영 모니터링 (자동)

매니저 도구로 매일 5분이면 확인:
- 앱 사용 통계 (오늘 몇 명 사용)
- 신규 가입자 (Vela 등록 필요 명단)
- 휴면 고객 자동 알림 (30일+ 미방문)
- 추천 통계

────────────────────────────────

사장님께 부탁드릴 일 (총 10분)

A. Firebase 보안 규칙 설정 (5분)

1. https://console.firebase.google.com 접속
2. specialmasterdj@gmail.com 로그인
3. kimchi-mart-order 프로젝트 클릭
4. 좌측 메뉴 → Realtime Database
5. 상단 Rules (규칙) 탭 클릭
6. 박스 내용 모두 지우기
7. 별도로 보내드리는 JSON 코드 붙여넣기
   (firebase-security-rules.json 파일)
8. 우측 상단 Publish (게시) 클릭
9. 끝

B. Blaze 플랜 전환 (5분)

1. 같은 Firebase Console 화면
2. 좌측 하단 "현재 플랜: Spark" 표시
3. Modify Plan / Upgrade 클릭
4. Blaze (Pay as you go) 선택
5. 신용카드 등록
6. 끝

────────────────────────────────

비용 대비 효과 (월 추정)

투자: 40~50만원

회수 가능성:
- 휴면 고객 회복 5천 명 × 30불 = 약 1천 5백만원
- 추천 신규 500명 × 30불 = 약 1천 5백만원
- K1 트라이얼 → 결제 전환 200명 = 약 25만원 회비
- 생일 보너스 재방문 = 약 1천만원

합계 잠재 매출: 1억원 이상
ROI: 200배 (보수적)

────────────────────────────────

결론

1. 5만 손님이 모두 사용해도 시스템은 멈추지 않습니다 (자동 확장)
2. 단, 보안은 오늘 안 막아야 합니다 (데이터 유출 위험)
3. 유료 전환은 이번 달 안에 (안 하면 한도 초과 시 멈춤)
